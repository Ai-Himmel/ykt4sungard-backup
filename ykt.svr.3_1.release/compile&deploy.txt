This document is a guide to compile ykt core bankgroud service.

System Environment Requirement:
  Redhat 5.x, must include 
  1)gcc 3.4.6  and dependencies rpms.
  2)oracle 10g.
  3)pcre 7.x
  you can make a virtuall machine os to do it.

  
steps:
1.create group and user.
    create a group named ykt.
    create a user named ykt.
	set user ykt's default home directory is /home/ykt ()
	set user ykt's default bash is /bin/bash

2.get release version files from svn by user ykt.
    (svnurl:http://bugzilla.kingstargroup.com/svn/ykt_svrsrc/branches/ykt.svr.3_1.release)
	run svn co http://bugzilla.kingstargroup.com/svn/ykt_svrsrc/branches/ykt.svr.3_1.release ykt at /home/ykt
	if downloaded all files, the directory structure will be :
		/home
			ykt
				ykt
					bin
					install
					lib
					log
					sbin
					sql_xxx
					src
					test

3.comiple project.
	1)login as ykt.
	2)chmod u+x ~/ykt/*.sh 
	3)run ~/ykt/compile.sh. if you want to clean and compile ,you can run "compile.sh clean"

4.create a timestamp file for making upgrade package.
	touch ~/ykt/upgrade_his.log

5.deploy project.
	1)cd ~/ykt
	2)run ykt_deploy.sh. it has three params.
		ykt_deploy.sh deploy_type(-a or -u) project_dir version
		in this env, all install package example:  ./ykt_deploy.sh -a /home/ykt/ykt 3.1
		             all upgrade package example:  ./ykt_deploy.sh -u /home/ykt/ykt 3.1
